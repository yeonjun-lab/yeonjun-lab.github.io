## 1. 왜 이 주제를 공부했는가

같은 양의 데이터를 처리해도  
순차 접근과 무작위 접근의 성능은 크게 다를 수 있다.  
이 차이는 C에서 특히 직접적으로 드러난다.

즉, 메모리 접근 패턴을 이해해야 성능 문서를 실제 코드 구조와 연결할 수 있다.

## 2. 정의

메모리 접근 패턴은 프로그램이 데이터를  
어떤 순서와 간격으로 읽고 쓰는지를 뜻한다.

## 3. 핵심 개념

### 3.1 연속 접근은 보통 더 유리하다

인접 데이터를 순서대로 읽으면 캐시와 하드웨어 예측이 잘 맞는다.

### 3.2 포인터 추적은 비용을 키울 수 있다

비연속 구조를 따라가면  
다음 데이터가 멀리 있을 가능성이 높다.

### 3.3 같은 알고리즘도 접근 순서에 따라 느껴지는 성능이 달라진다

연산량이 비슷해도  
데이터 배치와 접근 순서가 달라지면 병목이 달라진다.

## 4. 예시

배열을 앞에서부터 순서대로 읽는 루프와  
여러 노드를 포인터로 따라가며 읽는 루프는  
같은 횟수의 계산이라도 비용 모델이 다를 수 있다.

## 5. 헷갈리기 쉬운 부분

### 5.1 연산 수만 적으면 빠르다고 생각하는 문제

메모리 대기 비용이 더 클 수 있다.

### 5.2 자료구조 선택이 성능과 별개라고 보는 문제

자료구조는 접근 패턴을 사실상 결정한다.

### 5.3 캐시는 하드웨어 문서에서만 중요한 개념이라고 보는 문제

실제 코드 배치와 루프 설계에 직접 영향을 준다.

## 6. 어디서부터 직관과 실제 규칙이 갈리는가

### 6.1 "같은 데이터 양"과 "같은 메모리 비용"은 다른 말이다

어떤 순서로 읽는지가 실제 비용을 바꾼다.

### 6.2 포인터 기반 유연성과 성능은 같은 방향이 아닐 수 있다

추상화가 좋아도 메모리 지역성은 나빠질 수 있다.

### 6.3 성능은 계산식보다 데이터 이동에서 더 갈릴 수 있다

이동과 대기가 커지면 연산 자체는 작은 부분이 된다.

## 7. 실무 관점

### 7.1 접근 순서를 먼저 본다

순차 접근, 반복 접근, 무작위 접근을 구분한다.

### 7.2 자료구조와 배치를 함께 본다

배열, 구조체, 링크드 구조는 성능 모델이 다르다.

### 7.3 측정으로 확인한다

실제 병목이 메모리인지 확인하고 바꾸는 편이 안전하다.

## 8. 판단 체크리스트

- 지금 병목이 계산보다 메모리 접근에 있는지 확인했는가
- 데이터가 연속적인지, 포인터 추적이 많은지 보고 있는가
- 같은 양의 데이터와 같은 비용을 같은 뜻으로 보고 있지 않은가
- 자료구조 선택이 접근 패턴을 바꾼다는 점을 의식하고 있는가
- 캐시/지역성 문제를 측정과 함께 보고 있는가

## 정리

메모리 접근 패턴은 C 성능을 바꾸는 큰 축이다.  
핵심은 **무엇을 계산하느냐만이 아니라 데이터를 어떻게 읽느냐**다.
