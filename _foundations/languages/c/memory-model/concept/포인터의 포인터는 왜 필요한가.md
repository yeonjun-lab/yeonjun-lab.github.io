## 정의

포인터의 포인터는 "주소를 담는 변수" 자체의 주소를 다루기 위해 필요하다.  
즉 `int *p`가 어떤 `int` 객체의 주소를 담는다면, `int **pp`는 그 포인터 변수 `p`의 주소를 담는다.

C에서 이중 포인터가 필요한 이유는 단순히 별표가 하나 더 붙는 문법 때문이 아니다.  
핵심은 다음이다.

1. C의 함수 인자는 값으로 전달된다
2. 포인터도 하나의 값이다
3. 따라서 함수 안에서 포인터 변수 자체를 바꾸려면 그 포인터의 주소를 넘겨야 한다

즉 포인터의 포인터는 "한 번 더 역참조하는 어려운 문법"이 아니라,  
**포인터라는 값 자체를 수정하기 위한 도구**다.

---

## 핵심 개념

### 1. 포인터도 변수이고, 따라서 자기 자신의 주소를 가진다

예시:
```c
    int x = 10;
    int *p = &x;
```
여기서 `p`는 `x`를 가리키는 포인터다.  
즉 `p`의 값은 `x`의 주소다.

하지만 동시에 `p`도 메모리 어딘가에 저장된 변수다.  
따라서 `p` 자신도 주소를 가진다.

예시:
```c
    int **pp = &p;
```
이제 `pp`는 `p`의 주소를 담는다.

관계를 정리하면 다음과 같다.

- `x`는 정수 객체
- `p`는 `x`를 가리키는 포인터
- `pp`는 `p`를 가리키는 포인터

즉 이중 포인터는 "주소의 주소"를 표현한다.

---

### 2. 역참조 단계가 하나씩 늘어난다

예시:
```c
    int x = 10;
    int *p = &x;
    int **pp = &p;
```
각 표현식의 의미는 다음과 같다.

- `x` : 정수 값
- `&x` : `x`의 주소
- `p` : `x`의 주소를 담은 값
- `*p` : `p`가 가리키는 곳의 값, 즉 `x`
- `&p` : 포인터 변수 `p`의 주소
- `pp` : `p`의 주소를 담은 값
- `*pp` : `p`
- `**pp` : `x`

즉 별표 개수는 "몇 번 따라가야 실제 대상 값에 도달하는가"를 나타낸다.

---

### 3. 함수 안에서 포인터 자체를 바꾸려면 이중 포인터가 필요하다

이중 포인터가 왜 필요한지 가장 잘 드러나는 지점은 함수 인자 전달이다.

다음 코드를 보자.
```c
    void set_ptr(int *p) {
        static int x = 10;
        p = &x;
    }

    int main(void) {
        int *p = NULL;
        set_ptr(p);
    }
```
많은 초보자는 `set_ptr` 호출 후 `p`가 `x`를 가리킬 것이라고 생각한다.  
하지만 실제로는 아니다.

이유는 C의 함수 인자가 값으로 전달되기 때문이다.

- `main`의 `p`도 포인터 값
- `set_ptr`의 매개변수 `p`도 포인터 값
- 하지만 둘은 같은 변수가 아니다
- 함수 안에서 바뀌는 것은 복사본이다

즉 함수 안에서 `p = &x;`를 해도, 원래의 `p`는 바뀌지 않는다.

원래의 포인터 변수를 바꾸려면 그 포인터의 주소를 넘겨야 한다.

예시:
```c
    void set_ptr(int **pp) {
        static int x = 10;
        *pp = &x;
    }

    int main(void) {
        int *p = NULL;
        set_ptr(&p);
    }
```
이제 `set_ptr` 안의 `*pp`는 `main`의 `p` 자체를 의미한다.  
따라서 `*pp = &x;`는 원래 포인터 변수를 실제로 바꾼다.

이것이 이중 포인터의 가장 중요한 사용 이유다.

---

### 4. "대상을 바꾸는 것"과 "포인터를 바꾸는 것"은 다르다

이 지점에서 많은 혼동이 생긴다.

예시:
```c
    void set_value(int *p) {
        *p = 100;
    }
```
이 함수는 포인터가 가리키는 대상의 값을 바꾼다.

반면 아래는 다르다.
```c
    void set_pointer(int **pp) {
        static int x = 100;
        *pp = &x;
    }
```
이 함수는 포인터가 가리키는 대상을 바꾸는 것이 아니라,  
**포인터 변수 자체가 무엇을 가리킬지**를 바꾼다.

즉 다음 두 문장은 완전히 다르다.

- `*p = 100;` → 대상 값 수정
- `*pp = &x;` → 포인터 자체 수정

이 차이를 이해하지 못하면 이중 포인터는 계속 헷갈린다.

---

## 메모리 그림으로 보면 더 명확하다

예시:
```c
    int x = 10;
    int *p = &x;
    int **pp = &p;
```
메모리 개념 그림은 다음처럼 생각하면 된다.

- `x` 칸에는 정수 10이 들어 있다
- `p` 칸에는 `x`의 주소가 들어 있다
- `pp` 칸에는 `p`의 주소가 들어 있다

따라서

- `p`를 따라가면 `x`에 도달
- `pp`를 한 번 따라가면 `p`
- `pp`를 두 번 따라가면 `x`

즉 이중 포인터는 추상 문법이 아니라,  
메모리 안에 있는 "연결 단계가 하나 더 긴 구조"다.

---

## 언제 실제로 필요한가

### 1. 함수에서 포인터를 설정해야 할 때

가장 대표적인 경우다.

예시:
```c
    void allocate_int(int **pp) {
        *pp = malloc(sizeof(int));
        if (*pp != NULL) {
            **pp = 42;
        }
    }

    int main(void) {
        int *p = NULL;
        allocate_int(&p);
    }
```
여기서 함수는 단순히 값을 쓰는 것이 아니라,  
호출자 쪽 포인터가 어떤 동적 메모리를 가리키게 만들고 있다.

이런 상황에서는 `int **`가 필요하다.

---

### 2. 문자열 배열이나 인자 벡터를 다룰 때

예시:
```c
    int main(int argc, char **argv)
```
여기서 `argv`는 문자열 하나를 가리키는 포인터가 아니라,  
문자열들의 시작 주소들을 담은 배열과 연결된 구조를 다루기 위한 포인터다.

개념적으로 보면:

- 각 문자열은 `char *`
- 그런 문자열 포인터들이 여러 개 있다
- 그 집합을 다루기 위해 `char **`가 나온다

즉 `char **argv`는 "문자 하나를 두 번 따라가는 타입"이 아니라,  
**문자열 포인터들의 집합을 다루는 타입**으로 이해해야 한다.

---

### 3. 2차원처럼 보이는 동적 구조를 만들 때

예시:
```c
    int **matrix;
```
이 선언은 종종 "2차원 배열"처럼 소개되지만, 정확히는 **포인터들의 포인터**다.  
즉 진짜 연속된 2차원 배열과는 다르다.

예를 들어 각 행을 따로 할당하면:
```c
    matrix = malloc(rows * sizeof(int *));
    for (int i = 0; i < rows; i++) {
        matrix[i] = malloc(cols * sizeof(int));
    }
```
이 구조는 "행마다 별도 메모리 블록"인 형태다.  
따라서 `int **`는 언제나 진짜 2차원 배열과 같지 않다.

이 부분은 나중의 배열/포인터/다차원 배열 문서에서 다시 깊게 다뤄야 한다.

---

### 4. 연결 구조를 수정할 때

연결 리스트 같은 구조에서도 이중 포인터가 자주 나온다.

예시:
```c
    struct Node {
        int value;
        struct Node *next;
    };
```
헤드 포인터를 수정해야 하는 삽입/삭제 함수에서는 종종 `struct Node **head` 형태가 필요하다.

왜냐하면 함수가 리스트의 첫 노드를 바꾸려면,  
호출자의 헤드 포인터 자체를 수정해야 하기 때문이다.

즉 이중 포인터는 자료구조 구현에서도 매우 실전적이다.

---

## 자주 하는 오해

### 오해 1. 포인터의 포인터는 "포인터를 두 번 거쳐서 값 읽는 특수 문법"일 뿐이다

아니다.  
그것도 맞지만 핵심은 아니다.

진짜 핵심은 다음이다.

- 포인터도 값이다
- 포인터도 변수다
- 따라서 포인터 자신도 주소를 가진다
- 그 주소를 다루려면 이중 포인터가 필요하다

즉 이중 포인터는 포인터 변수 자체를 조작하기 위한 구조다.

---

### 오해 2. `int **`는 그냥 2차원 배열이다

아니다.  
`int **`는 포인터의 포인터일 뿐이다.

진짜 2차원 배열은 예를 들어 다음과 같이 선언된다.
```c
    int arr[3][4];
```
이것과 `int **`는 메모리 배치와 타입이 다르다.  
둘을 같은 것으로 보면 함수 인자, `sizeof`, 연속성, 캐시 친화성에서 전부 헷갈리게 된다.

---

### 오해 3. 포인터만 넘겨도 함수에서 포인터를 바꿀 수 있다

아니다.  
포인터를 넘기면 그 포인터 값의 복사본이 전달된다.

원래 포인터 변수를 바꾸려면 `&p`를 넘기고,  
매개변수는 `T **`로 받아야 한다.

---

### 오해 4. 별표 개수만 늘어나면 무조건 더 어려운 것이다

반은 맞고 반은 틀리다.  
어려운 이유는 문법이 아니라, **무엇을 수정하고 있는지**를 명확히 구분하지 않기 때문이다.

아래만 분명히 구분하면 된다.

- `p` : 주소 값
- `*p` : 대상 객체
- `pp` : 포인터의 주소 값
- `*pp` : 포인터 변수
- `**pp` : 최종 대상 객체

즉 별표는 난이도 표시가 아니라 **간접 참조 단계 수**다.

---

## `char **`가 특히 자주 나오는 이유

실무에서 이중 포인터를 가장 자주 보는 형태 중 하나가 `char **`다.

대표 예시는 다음과 같다.

- `argv`
- 문자열 배열
- 동적으로 여러 문자열을 관리하는 함수
- 토큰화 결과를 반환하는 함수

예를 들어 문자열 하나는 `char *`다.
```c
    char *s = "hello";
```
문자열 여러 개를 다루려면 각 문자열의 시작 주소가 여러 개 필요하다.
```c
    char *words[3];
```
이 배열이 표현식에서 decay되면 `char **`처럼 다뤄질 수 있다.

즉 `char **`는 C 문자열 처리와 매우 자주 연결된다.

---

## const와 결합하면 더 조심해야 한다

이중 포인터는 `const`와 결합하면 더 헷갈린다.

예시:
```c
    const char *p;
    const char **pp;
```
여기서는 각 단계마다 "무엇이 수정 가능한가"를 따로 봐야 한다.

- `const char *p` : 문자는 수정 불가, 포인터 변수 `p`는 바꿀 수 있음
- `const char **pp` : `pp`가 가리키는 포인터가 `const char *` 타입임

즉 이중 포인터는 단순히 별 하나 더 붙는 것이 아니라,  
타입 해석이 한 단계 더 중첩된다는 뜻이다.

이 부분은 나중의 `const는 무엇을 제한하는가` 문서와 직접 연결된다.

---

## 어디서부터 직관과 실제 간접 참조 규칙이 갈리는가

### 1. 포인터를 넘긴다고 해서 포인터 변수 자체가 공유되는 것은 아니다

가장 흔한 오해는 "포인터를 넘겼으니 참조처럼 동작하겠지"라는 생각이다.  
하지만 C는 여전히 값 전달 언어다.

즉 다음을 구분해야 한다.

- 포인터가 가리키는 대상 객체를 바꾸는가
- 포인터 변수 자체가 어디를 가리키는지 바꾸는가

첫 번째는 `T *`로도 가능하지만,  
두 번째는 포인터 변수의 주소가 필요하므로 `T **`가 필요하다.

### 2. `T **`는 "무조건 2차원"을 뜻하지 않는다

이중 포인터를 보자마자 2차원 배열처럼 이해하면 곧바로 막힌다.  
`T **`의 본질은 "두 단계 따라가면 T에 도달한다"이지,  
메모리가 연속된 2차원으로 놓였다는 뜻이 아니다.

즉 이 주제는 문법보다  
**간접 참조 단계와 데이터 구조 모양을 분리해서 보는 훈련**에 가깝다.

---

## 실무 관점

### 1. 동적 메모리 할당 API를 만들 때 필수다

호출자에게 새 메모리를 연결해 주는 함수는 보통 이중 포인터를 받는다.

예시:
```c
    int make_buffer(char **out) {
        *out = malloc(128);
        if (*out == NULL) {
            return -1;
        }
        return 0;
    }
```
이 함수는 문자열 내용을 바꾸는 것이 아니라,  
호출자 포인터가 새 버퍼를 가리키게 만든다.

이런 설계는 시스템 코드와 라이브러리 코드에서 매우 흔하다.

---

### 2. 자료구조 구현에서 자주 등장한다

- 연결 리스트 head 수정
- 트리의 루트 교체
- 해시 버킷 포인터 교체
- 동적 배열 재할당 결과 반영

즉 이중 포인터는 "드문 특수 문법"이 아니라,  
**포인터 기반 자료구조를 실제로 수정할 때 자주 쓰는 기본 도구**다.

---

### 3. 잘못 쓰면 버그도 강해진다

이중 포인터는 강력하지만 실수하면 위험하다.

예를 들어 다음을 혼동하면 문제가 생긴다.
```c
- `*pp = malloc(...)`
- `**pp = value`
```
첫 번째는 포인터를 설정하는 것이고,  
두 번째는 할당된 대상에 값을 쓰는 것이다.

이 둘을 혼동하면 NULL 역참조, 잘못된 쓰기, 미할당 메모리 접근 같은 버그가 바로 나온다.

---

## 판단 체크리스트

이중 포인터가 필요한지 판단할 때는 아래 순서가 가장 명확하다.

1. 함수가 "대상 값"을 바꾸려는가, "포인터 자체"를 바꾸려는가
2. 호출자 쪽 포인터 변수가 새 주소를 가리키게 해야 하는가
3. 지금 구조가 진짜 연속된 2차원 배열인가, 포인터 집합인가
4. `*pp`와 `**pp`가 각각 무엇을 뜻하는지 문장으로 설명 가능한가

즉 `T **`는 복잡한 문법이 아니라  
**값 전달 언어에서 포인터 값 자체를 수정하기 위해 필요한 최소 도구**로 읽는 편이 맞다.

## 정리

포인터의 포인터는 단순히 별표가 하나 더 붙은 어려운 문법이 아니다.  
그것은 **포인터 변수 자체의 주소를 다루기 위한 타입**이다.

핵심은 아래처럼 정리할 수 있다.

- 포인터도 하나의 값이다
- 포인터도 하나의 변수다
- 따라서 포인터도 자기 자신의 주소를 가진다
- 함수에서 원래 포인터를 바꾸려면 그 주소를 넘겨야 한다
- 그래서 `T **`가 필요하다

즉 이중 포인터는 C가 값 전달 언어라는 사실에서 자연스럽게 따라오는 구조다.

이 관점이 잡히면 다음 주제들도 훨씬 쉬워진다.

- 배열은 언제 포인터로 decay되는가
- 함수 인자는 왜 값으로 전달되는가
- 문자열 배열과 `char **argv`
- `int **`와 진짜 2차원 배열의 차이
- const가 포인터 단계마다 무엇을 제한하는가
