## 정의

C에서 문자와 문자열은 비슷해 보이지만 같은 것이 아니다.  
문자는 하나의 문자 코드 값을 뜻하고, 문자열은 여러 문자가 연속해서 배치되고 마지막에 널 문자 `'\0'`로 끝나는 문자 시퀀스다.

즉 다음은 전혀 다르다.

- `'A'` : 문자 하나
- `"A"` : 문자열
- `char c = 'A';` : 문자 하나를 담는 객체
- `char s[] = "A";` : 문자 두 개 `'A'`, `'\0'`를 담는 배열

이 차이를 구분하지 않으면 C의 문자열 모델, 배열 decay, 포인터, `sizeof`, 문자열 리터럴, 함수 인자 전달이 전부 흐려진다.

핵심은 다음이다.

1. 문자는 하나의 값이다
2. 문자열은 문자들의 연속된 시퀀스다
3. C의 문자열은 길이 정보 대신 널 종료 규칙에 의존한다
4. 문자와 문자열은 타입도, 메모리 구조도, 연산 방식도 다르다

즉 C에서 문자열은 별도의 내장 고급 객체가 아니라,  
**`char`들의 배열과 널 종료 규칙 위에 구축된 관례적 구조**다.

---

## 핵심 개념

### 1. 문자는 "문자 하나를 나타내는 값"이다

예시:
```
    char c = 'A';
```
여기서 `'A'`는 문자 하나를 뜻한다.  
즉 `c`는 단일 `char` 객체다.

이때 중요한 점은 다음이다.

- `c`는 배열이 아니다
- `c`는 문자열도 아니다
- `c`는 문자 코드 하나를 담는 객체다

즉 문자는 하나의 값으로 이해해야 한다.

---

### 2. 문자열은 "문자들의 연속 + 마지막 널 문자"다

예시:
```
    char s[] = "A";
```
이 선언은 문자 하나만 저장하는 것처럼 보이지만 실제로는 아니다.  
메모리에는 다음이 들어간다.

- `'A'`
- `'\0'`

즉 `"A"`는 길이 1의 문자열이지만, 저장에는 최소 2칸이 필요하다.

예시:
```
    char s[] = "ABC";
```
메모리에는 다음이 들어간다.

- `'A'`
- `'B'`
- `'C'`
- `'\0'`

즉 C에서 문자열은 "눈에 보이는 문자들"만으로 끝나지 않는다.  
문자열의 끝을 알리기 위한 널 종료 문자가 항상 핵심이다.

---

### 3. `'A'`와 `"A"`는 타입부터 다르다

이 부분이 가장 중요하다.

예시:
```
    'A'
    "A"
```
겉보기에는 비슷하지만 의미가 다르다.

- `'A'` : 문자 하나
- `"A"` : 문자열 리터럴

실무적으로 이해할 때는 아래처럼 잡는 것이 좋다.

- `'A'`는 문자 하나를 표현하는 값
- `"A"`는 `'A'`, `'\0'`로 이루어진 문자 시퀀스

즉 둘은 같은 "A"가 아니다.

예시:
```
    char c = 'A';      // 가능
    char *p = "A";     // 문자열 리터럴 위치를 가리킴
    char s[] = "A";    // 배열에 문자열 복사하여 초기화
```
셋 다 전혀 다른 메모리 모델이다.

---

### 4. C에는 진짜 "string 타입"이 없다

다른 언어에서는 문자열이 독립적인 고수준 타입인 경우가 많다.  
하지만 C에는 그런 내장 문자열 타입이 없다.

C에서 문자열은 보통 아래 둘 중 하나로 표현된다.

- `char[]`
- `char *`

하지만 둘은 같은 것이 아니다.

예시:
```
    char s1[] = "hello";
    char *s2 = "hello";
```
`s1`은 문자열 내용을 직접 담는 배열 객체다.  
반면 `s2`는 어떤 문자열 리터럴 위치를 가리키는 포인터다.

즉 둘 다 문자열처럼 사용할 수는 있지만,  
저장 방식과 수정 가능성은 다르다.

---

## 메모리 관점에서 보면 왜 다른가

### 1. 문자 하나는 객체 하나다

예시:
```
    char c = 'X';
```
이 경우 메모리에는 `char` 한 칸만 있으면 된다.

즉 문자 하나는 단일 객체다.

---

### 2. 문자열은 연속된 여러 칸이 필요하다

예시:
```
    char s[] = "XYZ";
```
이 경우 메모리에는 최소 네 칸이 필요하다.

- `s[0] = 'X'`
- `s[1] = 'Y'`
- `s[2] = 'Z'`
- `s[3] = '\0'`

즉 문자열은 단일 객체가 아니라,  
문자들이 연속 저장된 배열 구조다.

---

### 3. 널 문자가 문자열 경계다

C 문자열은 보통 길이 필드를 따로 저장하지 않는다.  
대신 `'\0'`를 만날 때까지를 문자열로 본다.

예시:
```
    char s[] = {'A', 'B', 'C', '\0'};
```
이것은 문자열이다.

반면 아래는 다르다.
```
    char s[] = {'A', 'B', 'C'};
```
이것은 문자 배열일 수는 있지만,  
널 종료가 없으므로 일반적인 C 문자열로 안전하게 다룰 수 없다.

즉 C에서 문자열은 단순 문자 배열이 아니라  
**널 종료 규약을 만족하는 문자 배열**이다.

---

## `char`, `char[]`, `char *`는 왜 각각 다르게 봐야 하는가

### 1. `char`

예시:
```
    char c = 'A';
```
문자 하나를 담는 객체다.

---

### 2. `char[]`

예시:
```
    char s[] = "ABC";
```
문자열 내용을 직접 담는 배열이다.  
배열 안에 실제 문자가 들어 있다.

즉 `s`는 저장공간을 소유한다.

---

### 3. `char *`

예시:
```
    char *p = "ABC";
```
문자열 자체를 담는 것이 아니라,  
어딘가에 있는 문자 시퀀스의 시작 위치를 가리킨다.

즉 `p`는 저장공간을 직접 소유하는 것이 아니라 주소를 담는 포인터다.

이 차이는 매우 중요하다.

---

## 왜 `sizeof`에서 차이가 드러나는가

예시:
```
    char c = 'A';
    char s[] = "ABC";
    char *p = "ABC";
```
각각을 보면:

- `sizeof(c)` : 문자 하나 크기
- `sizeof(s)` : 배열 전체 크기, 보통 4
- `sizeof(p)` : 포인터 크기

즉 문자열처럼 보여도 `char[]`와 `char *`는 전혀 다르다.

`"ABC"`는 눈으로는 3글자지만,  
`char s[] = "ABC";`에서는 널 문자까지 포함되어 총 4칸이 된다.

---

## 문자열 리터럴은 왜 특별한가

예시:
```
    "hello"
```
이것은 단순한 글자 묶음이 아니라 문자열 리터럴이다.  
문자열 리터럴은 프로그램 안 어딘가에 저장되는 문자 시퀀스이며, 끝에는 `'\0'`가 붙는다.

실무적으로는 아래처럼 이해하면 좋다.

- 문자열 리터럴은 고정된 문자 시퀀스다
- 포인터는 그 시작 위치를 가리킬 수 있다
- 배열은 그 내용을 복사해 초기화할 수 있다

예시:
```
    char *p = "hello";
    char s[] = "hello";
```
첫 줄에서는 포인터가 리터럴 위치를 가리킨다.  
둘째 줄에서는 배열에 내용이 복사되어 초기화된다.

즉 둘은 결과적으로 비슷하게 보여도, 메모리 소유 방식이 다르다.

---

## 수정 가능성에서 왜 차이가 나는가

예시:
```
    char s[] = "hello";
    s[0] = 'H';   // 가능
```
여기서는 `s`가 실제 배열 객체이므로 원소 수정이 가능하다.

반면 아래는 위험하다.
```
    char *p = "hello";
    p[0] = 'H';   // 수정 시도는 문제를 일으킬 수 있음
```
왜냐하면 `p`는 문자열 리터럴 위치를 가리키고 있고,  
문자열 리터럴은 수정 가능한 배열처럼 다뤄서는 안 되기 때문이다.

즉 문자열처럼 보인다고 해서 다 같은 메모리가 아니다.

---

## 문자 배열과 문자열은 왜 구분해야 하는가

모든 문자 배열이 문자열은 아니다.

예시:
```
    char a[3] = {'A', 'B', 'C'};
```
이것은 문자 배열이다.  
하지만 널 종료가 없으므로 일반적인 C 문자열 함수에 넘기면 위험하다.

반면:
```
    char b[4] = {'A', 'B', 'C', '\0'};
```
이것은 문자열로 다룰 수 있다.

즉 문자열은 문자 배열의 한 종류이지,  
모든 문자 배열이 문자열인 것은 아니다.

---

## 함수 인자에서 왜 더 헷갈리는가

예시:
```
    void print_str(char *s) {
        printf("%s\n", s);
    }
```
이 함수는 `char *`를 받는다.  
그런데 아래처럼 배열도 넘길 수 있다.
```
    char s[] = "hello";
    print_str(s);
```
왜 가능한가?  
배열이 함수 호출 문맥에서 decay하기 때문이다.

즉 배열 `s`는 전달 시 `char *`처럼 쓰인다.  
하지만 그렇다고 배열과 포인터가 같은 것은 아니다.

이 때문에 C의 문자열 처리는 문법상 간단해 보여도,  
실제로는 배열 decay 규칙 위에 서 있다.

---

## 자주 하는 오해

### 오해 1. `'A'`와 `"A"`는 같은 것이다

아니다.

- `'A'`는 문자 하나
- `"A"`는 문자열

메모리 구조와 사용 방식이 다르다.

---

### 오해 2. 모든 문자 배열은 문자열이다

아니다.  
널 종료가 있어야 일반적인 C 문자열이다.

---

### 오해 3. `char *`가 곧 문자열 그 자체다

아니다.  
`char *`는 문자열 시작 위치를 가리킬 수 있는 포인터일 뿐이다.

문자열 내용을 직접 소유하는 것은 `char[]`일 수 있다.

---

### 오해 4. `char s[] = "abc"`와 `char *s = "abc"`는 완전히 같다

아니다.

- 전자는 배열에 내용이 저장된다
- 후자는 포인터가 문자열 리터럴을 가리킨다

수정 가능성과 `sizeof` 결과도 달라진다.

---

### 오해 5. 문자열 길이는 배열 크기와 같다

항상 그렇지 않다.  
배열 크기에는 널 문자 공간이 포함될 수 있고,  
포인터는 길이 정보를 직접 갖지 않는다.

즉 문자열 길이와 저장 공간 크기는 같은 개념이 아니다.

---

## 어디서부터 문자 값, 문자열 표현, 저장공간 소유가 갈리는가

### 1. 문자는 값 하나이고 문자열은 규칙을 가진 시퀀스다

`'A'`와 `"A"`가 비슷해 보이기 때문에  
문자 하나를 담은 문자열과 문자 값 하나를 같은 것으로 느끼기 쉽다.  
하지만 문자는 단일 값이고, 문자열은 `char` 시퀀스와 널 종료 규칙 위에 성립한다.

즉, 문자열은 문자 여러 개의 단순 모음이 아니라  
종료 규칙까지 포함한 표현 모델이다.

### 2. `char *`는 문자열 자체가 아니라 문자열에 도달하는 경로일 수 있다

처음 배우는 사람은 `char *`를 보면 곧바로 문자열이라고 말하기 쉽다.  
하지만 포인터는 시작 위치를 가리킬 뿐이고,  
실제 저장공간을 소유하는 객체인지, 읽기 전용 리터럴을 가리키는지,  
수정 가능한 버퍼를 가리키는지는 별도 정보다.

즉, 포인터 타입과 문자열 소유 모델은 분리해서 봐야 한다.

### 3. 문자 배열과 문자열은 겹칠 수 있지만 같은 개념은 아니다

문자 배열 안에 널 종료가 들어 있으면 문자열처럼 쓸 수 있다.  
하지만 모든 문자 배열이 문자열은 아니고,  
모든 문자열 표현이 배열 객체 자체를 소유하는 것도 아니다.

즉, “문자 배열”은 저장공간 관점의 말이고  
“문자열”은 해석 규칙 관점의 말이다.

### 4. 문자열 리터럴은 값처럼 보여도 저장공간 정책이 따로 있다

문자열 리터럴은 소스에 그냥 적혀 있으니  
즉석 값처럼 느껴질 수 있다.  
하지만 실제로는 특정 저장공간에 놓인 문자 시퀀스로 취급되고,  
수정 가능성도 일반 배열 객체와 다르게 봐야 한다.

즉, 리터럴은 문법 표기와 런타임 저장 모델을 구분해서 이해해야 한다.

### 5. 문자열 길이, 배열 크기, 포인터 크기는 서로 다른 질문이다

문자열을 다루다 보면 길이와 크기를 한 단어로 섞어 말하기 쉽다.  
하지만 실제로는 다음이 다르다.

- 문자열 의미 길이
- 배열이 확보한 저장공간 크기
- 포인터 객체 크기

즉, 문자열 처리 버그의 상당수는  
이 세 질문을 하나로 뭉뚱그리면서 생긴다.

---

## 실무 관점

### 1. 문자열 버그의 출발점 대부분이 여기서 시작된다

다음 같은 실수가 매우 흔하다.

- 널 종료를 빼먹음
- 배열 크기와 문자열 길이를 혼동함
- 문자열 리터럴 수정 시도
- `char *`와 `char[]`를 같은 것으로 취급함
- 버퍼 크기보다 긴 문자열 복사

즉 문자와 문자열의 구분은 단순 이론이 아니라  
문자열 버그를 막는 가장 기초적인 기준이다.

---

### 2. API 설계에서 소유권과 수정 가능성을 결정한다

예를 들어 함수 인자가 다음 중 무엇인지에 따라 의미가 달라진다.

- `const char *` : 읽기 전용 문자열 입력
- `char *` : 수정 가능한 버퍼
- `char []` 표기 매개변수 : 실제로는 포인터이지만 버퍼라는 의도를 드러냄

즉 문자열을 넘길 때는 단순히 "글자 전달"이 아니라,  
누가 저장공간을 소유하고 누가 수정할 수 있는가까지 함께 설계해야 한다.

---

### 3. 바이너리 데이터와 텍스트 데이터를 구분하는 출발점이 된다

널 종료 규칙은 텍스트 문자열 처리에는 편리할 수 있다.  
하지만 바이너리 데이터에는 적합하지 않을 수 있다.

예를 들어 중간에 `'\0'`가 들어갈 수 있는 데이터는  
일반적인 C 문자열처럼 다루면 잘린다.

즉 문자열 모델을 제대로 이해해야  
텍스트와 바이너리 데이터를 구분해서 처리할 수 있다.

---

### 4. 이후 주제와 직접 연결된다

이 문서를 이해하면 다음 문서들이 자연스럽게 이어진다.

- 문자열은 왜 특별 취급되는가
- 함수 인자는 왜 값으로 전달되는가
- `char **argv`는 무엇을 의미하는가
- `sizeof`는 무엇을 알려주고 무엇을 숨기는가
- 버퍼 오버플로우는 어떻게 생기는가

즉 문자와 문자열 구분은 문자열 파트의 출발점이다.

---

## 판단 체크리스트

- 문자 값 하나와 널 종료 규칙을 가진 문자열 표현을 구분하고 있는가
- `char *`를 문자열 그 자체가 아니라 문자열에 도달하는 경로로 볼 수 있는가
- 문자 배열과 문자열을 저장공간 관점과 해석 규칙 관점으로 나눠 설명할 수 있는가
- 문자열 리터럴의 저장공간 정책과 수정 가능성을 일반 배열과 구분하고 있는가
- 문자열 길이, 배열 크기, 포인터 크기를 같은 질문으로 섞고 있지 않은가

---

## 정리

C에서 문자는 문자 하나를 나타내는 값이고,  
문자열은 `char`들의 연속과 마지막 널 문자 `'\0'`로 이루어진 구조다.

핵심은 다음이다.

- `'A'`와 `"A"`는 다르다
- `char`, `char[]`, `char *`는 다르다
- 모든 문자 배열이 문자열은 아니다
- 문자열은 널 종료 규칙 위에 성립한다
- 문자열 리터럴과 배열은 저장 방식이 다르다

즉 C에서 문자열을 제대로 이해하려면  
"글자들의 묶음" 정도로 생각해서는 안 되고,  
**문자 배열, 널 종료, 포인터, decay, 저장공간 소유 방식까지 함께 봐야 한다.**

이 관점이 잡히면 C 문자열은 갑자기 쉬워지지는 않아도,  
적어도 왜 자꾸 헷갈리는지는 명확해지기 시작한다.
