## 1. 왜 이 주제를 공부했는가

C를 배우는 대부분의 사람이 가장 크게 흔들리는 지점은 포인터다.  
포인터를 처음 보면 낯설고, 문법도 이상해 보이며, 실수하면 프로그램이 바로 터질 수도 있다.  
그래서 많은 사람이 포인터를 “C의 어려운 특수 문법” 정도로 받아들인다.

하지만 이 시각은 절반만 맞다.  
포인터가 어려운 이유는 포인터만 따로 이상해서가 아니다.  
그보다 먼저 알아야 할 개념, 즉 **주소**, **메모리 객체**, **값과 위치의 차이**, **수명**, **역참조**를 충분히 이해하지 못한 상태에서 포인터를 만나기 때문에 어렵게 느껴지는 것이다.

사실 포인터의 핵심은 복잡하지 않다.

- 포인터는 값을 저장한다.
- 그 값은 일반 정수값이 아니라 **주소**다.
- 그 주소를 통해 다른 객체에 접근할 수 있다.

즉, 포인터는 마법이 아니라 **주소를 저장하는 변수**다.

이 관점이 잡히면 다음 주제들이 한 번에 연결된다.

- 배열
- 문자열
- 함수 인자 전달
- 동적 메모리
- 구조체 기반 자료구조
- 연결 리스트, 트리, 그래프
- 시스템 프로그래밍
- 메모리 오류 디버깅

반대로 이 관점이 없으면, 포인터는 계속 외워야 하는 이상한 규칙처럼 보인다.

예를 들어 다음 코드를 보자.

```c
int x = 10;
int *p = &x;
```

겉으로는 짧다.  
하지만 이 두 줄 안에는 C의 핵심이 거의 다 들어 있다.

- `x`는 정수 객체다.
- `&x`는 그 객체의 주소다.
- `p`는 그 주소를 저장하는 변수다.
- `*p`를 통해 다시 `x`에 접근할 수 있다.

즉, 포인터는 별도 세계가 아니라 **이미 본 변수·주소·메모리 개념의 연장선**이다.

이 문서는 포인터를 처음 정의하고, 주소를 저장하는 값이라는 본질을 분명히 하며, 이후 배열·문자열·동적 메모리로 안정적으로 넘어가기 위한 기준점을 만들기 위해 작성한다.

## 2. 정의

### 2.1 포인터는 주소를 저장하는 값이다

포인터(pointer)는 어떤 객체가 위치한 **메모리 주소를 저장하는 값**이다.

즉, 포인터 자체도 값이고, 포인터 변수도 변수다.  
다만 일반 정수형 변수처럼 숫자 10, 20, 30 같은 값을 저장하는 대신,  
다른 객체가 놓인 위치를 저장한다는 점이 다르다.

예를 들어:

```c
int x = 10;
int *p = &x;
```

여기서 `p`는 10을 저장하는 것이 아니다.  
`x`가 메모리 어디에 있는지를 나타내는 주소를 저장한다.

### 2.2 포인터 변수와 가리키는 대상은 다르다

포인터를 이해할 때 가장 중요한 구분 중 하나는 이것이다.

- 포인터 변수 자체
- 포인터가 가리키는 대상 객체

위 예시에서:

- `p`는 포인터 변수다.
- `p` 안에 저장된 값은 주소다.
- 그 주소가 가리키는 대상은 `x`다.

즉, `p`와 `x`는 같은 것이 아니다.  
`p`는 `x`를 가리킬 뿐이다.

### 2.3 `&`는 주소를 얻는 연산자다

`&x`는 `x`의 주소를 의미한다.  
즉, 어떤 객체가 메모리 어디에 있는지를 알고 싶을 때 `&`를 사용한다.

예를 들어:

```c
int x = 10;
int *p = &x;
```

여기서 `&x`는 `x`라는 객체의 위치를 나타낸다.

### 2.4 `*`는 가리키는 대상을 따라가는 연산자다

포인터 안에는 주소가 들어 있다.  
그 주소를 따라가서 실제 객체에 접근하는 것을 **역참조(dereference)** 라고 한다.

예를 들어:

```c
int x = 10;
int *p = &x;
int y = *p;
```

여기서 `*p`는 “p가 가리키는 곳에 있는 값”이라는 뜻이다.  
즉, `*p`는 결국 `x`의 값을 읽는 것과 같은 의미가 된다.

### 2.5 포인터는 특정 타입과 함께 존재한다

포인터는 단순히 “주소만 저장하는 값”이 아니다.  
C에서 포인터는 보통 **어떤 타입의 대상을 가리키는가**와 함께 정의된다.

예를 들어:

```c
int *p;
char *q;
double *r;
```

이들은 모두 포인터지만, 서로 다른 타입의 객체를 가리킨다.

즉, 포인터는 “주소 저장 + 그 주소를 어떤 형식으로 해석할 것인가”를 함께 가진다.

### 2.6 null pointer는 아무 유효한 객체도 가리키지 않는 포인터다

포인터가 항상 어떤 유효한 객체를 가리키는 것은 아니다.  
아무 대상도 가리키지 않는 특별한 상태를 표현하기 위해 null pointer를 쓴다.

예를 들어:

```c
int *p = NULL;
```

이 경우 `p`는 어떤 유효한 `int` 객체도 가리키지 않는다.

즉, 포인터는 “항상 뭔가를 가리키는 값”이 아니라,  
때로는 의도적으로 아무것도 가리키지 않도록 설정할 수도 있다.

## 3. 핵심 개념

### 3.1 포인터도 변수다

초보자는 포인터를 너무 특별하게 생각하는 경향이 있다.  
하지만 포인터도 결국 변수다.

즉, 포인터 변수는 다음을 가진다.

- 타입
- 값
- 주소
- 수명
- scope

예를 들어:

```c
int x = 10;
int *p = &x;
```

여기서 `p`도 변수이므로 자기 자신의 주소도 가진다.  
즉, `p`는 `x`를 가리키는 변수이지, 스스로 사라지는 개념이 아니다.

### 3.2 포인터의 값은 “주소”다

일반 변수가 정수값이나 문자값을 저장하듯, 포인터는 주소를 저장한다.  
즉, 포인터의 본질은 “주소를 값으로 다루게 해주는 것”이다.

이 점이 중요한 이유는, C에서 주소도 결국 프로그램 안에서 전달하고 저장하고 비교할 수 있는 데이터라는 뜻이기 때문이다.

### 3.3 역참조는 주소를 통해 대상 객체에 접근하는 것이다

포인터 안의 주소만 보는 것과, 그 주소가 가리키는 대상 객체를 읽는 것은 다르다.

예를 들어:

```c
int x = 10;
int *p = &x;
```

- `p` → 주소
- `*p` → 그 주소가 가리키는 객체의 값

즉, 포인터는 두 층으로 봐야 한다.

1. 포인터 변수 자체
2. 포인터가 가리키는 대상

이 구분이 무너지면 포인터 이해가 무너진다.

### 3.4 포인터를 통해 원본 객체를 바꿀 수 있다

포인터는 단순 조회만 하는 것이 아니다.  
가리키는 대상 객체를 수정할 수도 있다.

예를 들어:

```c
int x = 10;
int *p = &x;
*p = 20;
```

이 경우 `*p = 20;`은 결국 `x = 20;`과 같은 효과를 낸다.  
즉, 포인터는 **원본 객체에 간접적으로 접근하는 통로**다.

### 3.5 포인터는 값 복사와 다른 접근 방식을 만든다

일반 대입은 보통 값을 복사한다.

```c
int a = 10;
int b = a;
```

이 경우 `b`를 바꿔도 `a`는 그대로다.

반면 포인터를 넘기면 원본 객체의 주소를 공유하는 효과가 생긴다.

즉, 포인터는 단순 값 복사 모델만으로는 할 수 없는 “간접 접근”을 가능하게 한다.  
그래서 함수 호출, 자료구조, 메모리 관리에서 매우 중요하다.

### 3.6 포인터는 수명 문제와 직결된다

포인터는 강력하지만, 가리키는 대상 객체의 수명에 의존한다.

예를 들어 어떤 포인터가 지역 변수의 주소를 저장하고 있다면,  
그 지역 변수의 lifetime이 끝난 뒤에는 그 포인터는 더 이상 안전하지 않다.

즉, 포인터는 단순 문법이 아니라 **객체 수명 추적 능력**과 함께 다뤄야 한다.

## 4. 예시

### 4.1 가장 기본적인 포인터 예시

다음 코드를 보자.

```c
int x = 10;
int *p = &x;
```

이 코드를 층별로 읽으면 다음과 같다.

- `x`는 정수 객체다.
- `&x`는 `x`의 주소다.
- `p`는 그 주소를 저장하는 포인터 변수다.

즉, `p`는 `x`를 직접 저장하는 것이 아니라 `x`의 위치를 저장한다.

### 4.2 역참조 예시

다음 코드를 보자.

```c
int x = 10;
int *p = &x;
printf("%d\n", *p);
```

여기서 `*p`는 “p가 가리키는 곳의 값”이다.  
즉, `*p`는 결국 `x`의 값을 읽는다.

따라서 출력 결과는 10이다.

### 4.3 포인터를 통한 수정 예시

다음 코드를 보자.

```c
int x = 10;
int *p = &x;
*p = 20;
```

이 경우 `*p = 20;`은 `x`를 직접 수정한 것과 같다.  
즉, 포인터는 대상을 간접적으로 변경할 수 있는 경로다.

이것이 중요한 이유는, 함수 안에서도 원본 객체를 수정할 수 있게 해주기 때문이다.

### 4.4 포인터 변수 자체도 자기 주소를 가진다는 예시

다음 코드를 보자.

```c
int x = 10;
int *p = &x;
printf("%p\n", (void*)&x);
printf("%p\n", (void*)&p);
```

여기서 `&x`와 `&p`는 다르다.

- `&x`는 `x`라는 객체의 주소
- `&p`는 `p`라는 포인터 변수 자체의 주소

즉, 포인터는 대상의 주소를 저장하는 변수일 뿐이며,  
그 포인터 변수 자신도 메모리 안에 존재하는 하나의 객체다.

### 4.5 null pointer 예시

다음 코드를 보자.

```c
int *p = NULL;
```

이 경우 `p`는 유효한 객체를 가리키지 않는다.  
따라서 바로 `*p`를 하면 안 된다.

즉, 포인터는 “무조건 역참조해도 되는 값”이 아니라,  
현재 유효한 대상을 가리키는지 먼저 확인해야 하는 값일 수도 있다.

## 5. 헷갈리기 쉬운 부분

### 5.1 포인터 변수와 가리키는 값을 혼동하는 문제

다음 코드를 보자.

```c
int x = 10;
int *p = &x;
```

여기서 초보자는 `p`가 10을 저장한다고 착각하기 쉽다.  
하지만 `p`는 10이 아니라 **x의 주소**를 저장한다.

10을 읽고 싶다면 `*p`를 봐야 한다.

즉, `p`와 `*p`는 전혀 다르다.

### 5.2 선언의 `*`와 역참조의 `*`를 같은 것으로 느끼는 문제

`*`는 포인터 문맥에서 두 가지 모습으로 보인다.

선언에서:

```c
int *p;
```

이건 “p는 int를 가리키는 포인터”라는 뜻이다.

사용에서:

```c
*p = 20;
```

이건 “p가 가리키는 대상에 접근한다”는 뜻이다.

겉모양은 같지만 역할은 문맥에 따라 다르다.  
이 점이 초반에 꽤 헷갈린다.

### 5.3 포인터를 마법처럼 보는 문제

포인터는 어려워 보이지만, 실제로는 단순하다.

- 주소를 저장한다.
- 주소를 따라가서 대상을 읽거나 쓴다.

즉, 포인터는 메모리 모델 위에서 보면 매우 자연스러운 개념이다.  
이걸 별도 마법 규칙처럼 외우면 계속 불안정해진다.

### 5.4 null pointer를 그냥 “빈 값” 정도로 가볍게 보는 문제

null pointer는 아무 대상도 가리키지 않는 포인터다.  
즉, 역참조할 수 없는 상태다.

따라서 다음은 위험하다.

```c
int *p = NULL;
*p = 10;
```

null pointer는 단순히 값이 없다는 뜻이 아니라,  
그 포인터를 통해 유효한 객체에 접근할 수 없다는 의미다.

### 5.5 포인터가 가리키는 대상의 수명을 잊는 문제

다음 코드는 매우 위험하다.

```c
int* bad(void) {
    int x = 10;
    return &x;
}
```

문제는 `&x`가 문법적으로 틀려서가 아니다.  
`x`의 lifetime이 함수 종료와 함께 끝나기 때문이다.

즉, 포인터는 “어디를 가리키는가”뿐 아니라 “그 대상이 아직 살아 있는가”까지 함께 봐야 한다.

## 6. 실무 관점

### 6.1 함수가 원본 데이터를 바꾸게 할 수 있다

포인터는 함수에 원본 객체의 주소를 넘길 수 있게 해준다.  
그래서 함수 안에서 caller의 데이터를 직접 수정할 수 있다.

예를 들어 값만 넘기면 복사본을 다루게 되지만,  
포인터를 넘기면 원본 객체에 접근할 수 있다.

이것은 C에서 매우 중요한 패턴이다.

### 6.2 배열, 문자열, 동적 메모리와 직접 연결된다

포인터를 이해하지 못하면 다음 주제를 끝까지 안정적으로 이해하기 어렵다.

- 배열 이름의 의미
- 문자열 처리
- `malloc` 반환값
- 구조체 연결
- 연결 리스트와 트리

즉, 포인터는 하나의 독립 주제가 아니라 **C의 연결 축**이다.

### 6.3 메모리 오류를 읽는 능력의 출발점이다

다음 같은 문제는 대부분 포인터와 관련된다.

- 잘못된 역참조
- null pointer dereference
- dangling pointer
- 잘못된 주소 접근
- use-after-free

즉, 포인터를 이해한다는 것은 단순히 코드를 쓰는 능력만이 아니라,  
메모리 오류를 해석하는 능력을 갖는다는 뜻이기도 하다.

### 6.4 시스템 수준 사고를 가능하게 한다

포인터는 주소를 값처럼 다루게 해준다.  
이것은 시스템 프로그래밍, 메모리 매핑, 버퍼 처리, 저수준 인터페이스에서 매우 중요하다.

즉, 포인터는 C가 왜 시스템과 가까운 언어인지 보여주는 대표적인 기능이다.

---

## 7. 다음에 볼 것

### 7.1 포인터 타입은 왜 중요한가

포인터는 주소를 저장하지만, 단순히 아무 주소나 저장하는 값이 아니다.  
어떤 타입의 대상을 가리키는지에 따라 해석과 연산이 달라진다.

### 7.2 배열과 포인터는 왜 비슷하면서 다른가

배열은 포인터와 매우 밀접하지만, 둘은 같은 것이 아니다.  
이 차이를 정확히 이해해야 `sizeof`, 함수 인자, 문자열 처리를 안정적으로 볼 수 있다.

### 7.3 문자열 리터럴과 char 배열의 차이

문자열은 `char *`와 `char[]` 형태로 자주 보인다.  
포인터를 이해한 뒤 이 둘의 차이를 보면 훨씬 덜 헷갈린다.

### 7.4 스택과 힙의 차이

포인터는 스택 객체도 가리킬 수 있고 힙 객체도 가리킬 수 있다.  
따라서 포인터 다음에는 객체 수명과 저장 영역 차이를 보는 것이 자연스럽다.

## 정리

포인터는 C에서 가장 중요한 개념 중 하나다.  
하지만 포인터의 본질은 복잡하지 않다.

핵심은 다음과 같다.

- 포인터는 주소를 저장하는 값이다.
- 포인터 변수 자체와 가리키는 대상은 다르다.
- `&`는 주소를 얻는다.
- `*`는 가리키는 대상에 접근한다.
- 포인터를 통해 원본 객체를 간접적으로 읽고 쓸 수 있다.
- 포인터는 대상 객체의 수명에 의존하므로 lifetime을 함께 봐야 한다.
- null pointer는 아무 유효한 객체도 가리키지 않는다.

즉, 포인터를 제대로 이해한다는 것은  
“이상한 문법을 외운다”가 아니라  
**주소를 값처럼 다루고, 그 주소를 통해 객체에 간접 접근할 수 있다는 구조를 이해한다**는 뜻이다.

이 관점이 잡히면 이후 배열, 문자열, 동적 메모리, 자료구조, 시스템 프로그래밍으로 훨씬 자연스럽게 이어질 수 있다.
